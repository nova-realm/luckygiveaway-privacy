# Luckygiveaway Privacy Policy

**Last updated:** July 23, 2026

This Privacy Policy explains what information Luckygiveaway ("the Bot," "we," "us") collects when it is added to a Discord server, how that information is used, and the choices available to users and server administrators.

## 1. Information We Collect

When the Bot is used to create or manage a giveaway, we may collect and store the following:

- **Server (Guild) ID** — to associate giveaways with the correct server.
- **Channel ID and Message ID** — to locate and update the giveaway message.
- **User IDs of giveaway hosts** — the person who created the giveaway.
- **User IDs of giveaway entrants** — members who enter a giveaway.
- **Giveaway configuration data** — prize description, duration, number of winners, and any role or eligibility requirements set by the host.
- **User IDs of selected winners** — to track results and support rerolls.

We do **not** collect or store:
- Message content unrelated to giveaway commands.
- Presence data (online/idle/offline status).
- Direct message content, unless a user directly messages the Bot to interact with a giveaway feature that requires it.

## 2. How We Use This Information

We use the data described above solely to:
- Operate and display giveaways (entries, countdowns, winner selection).
- Verify entrant eligibility (e.g., required roles, minimum account age, or join-date restrictions) to reduce abuse from alternate accounts.
- Allow server staff to manage, end, reroll, or review past giveaways.
- Announce and, where applicable, notify winners.

We do not sell user data, and we do not use any collected data to train machine learning or AI models.

## 3. Message Content

The Bot supports both slash commands and legacy prefix commands (e.g., `!gstart`). When a prefix command is used, the Bot reads the content of that specific message to extract command parameters (such as prize, duration, and winner count). This parsed configuration data is stored as described in Section 1; the raw message text itself is not retained after processing.

Users who prefer not to have any message content read by the Bot can use slash commands exclusively, which do not require the Message Content intent.

## 4. Data Retention

Giveaway data is retained for as long as the Bot remains in a server, in order to support features like rerolling past giveaways. ## 5. Data Sharing

We do not share, sell, or disclose user data to third parties, except:
- Where required by law or valid legal process.
- To protect the rights, safety, or property of our users, our service, or the public.

## 6. Data Storage and Security

Data is stored in a private database that is not publicly accessible. We take reasonable technical measures to protect stored data from unauthorized access, alteration, or disclosure.
